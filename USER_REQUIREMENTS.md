# 学习心理诊断工具 - 用户需求文档

## 产品定位

一个基于AI的学习心理诊断工具，帮助学生通过3轮温暖的心理咨询对话，深入了解自己的学习心理状态，最后生成专业的心理诊断报告。

## 用户场景

学生在学习中遇到三种常见的心理困境：

1. **失意之径** - 努力后却考砸了，感到失落和困惑
2. **深谷挑战** - 深夜遇到难题卡住，感到无助和困顿
3. **意志荒漠** - 学习动力消退，想放弃去刷视频

## 核心流程

### 第一步：首页选择场景
用户进入应用，看到三个场景卡片，选择符合自己当前心理状态的场景。

### 第二步：进入对话
用户进入对话页面，与AI心理咨询师进行3轮对话：
- AI提出开放式问题
- 用户详细回答
- AI基于回答提出下一个问题
- 重复3次

### 第三步：生成报告
完成3轮对话后，系统生成心理诊断报告，包括：
- 四个心理维度的评分（归因风格、自我效能感、认知负荷、元认知）
- 雷达图展示
- AI的专业分析和建议

## 页面结构

### 首页
- 顶部：品牌信息和用户信息
- 中间：三个场景卡片（可点击进入对话）
- 底部：四个心理维度的介绍

### 对话页面
- 顶部：返回按钮、场景名称、进度显示
- 中间：对话历史（微信风格气泡）
- 底部：用户输入框和提交按钮

### 报告页面
- 雷达图：四个维度的分值可视化
- 分值卡片：具体数值展示
- 分析文字：AI生成的专业分析
- 建议：个性化的改进建议

## 关键需求

1. **对话要自然温暖** - AI的问题要同情、专业、开放式，能深入了解用户心理

2. **评分要合理** - 根据用户的回答内容和长度，动态评估四个心理维度

3. **报告要专业** - 最后的诊断报告要有数据支撑，分析要有深度

4. **体验要流畅** - 对话页面简洁，只显示对话框，不要分散注意力

5. **要有容错机制** - 如果API调用失败，要能自动降级使用预设问题继续对话

## 逻辑闭环

用户选择场景 → 进入对话 → 完成3轮对话 → 系统评分 → 生成报告 → 用户查看诊断结果

整个过程通过AI驱动的对话来收集用户的心理信息，然后基于这些信息进行评分和分析，最后输出专业的诊断报告。
